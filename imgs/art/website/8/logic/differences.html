Quantum Vs Classical Logic

Quantum logic has essentially many differences than classical logic. Besides, if this was not the case then we might not need a brand-new kind of logic to describe quantum theory.

Before explaining some significant differences among these two kinds of logic, we should consider their origin. The main causes of differences are:

  • The probabilistic nature of quantum mechanics and the uncertainty involved in extracting results.
  • The fact that when a measurement is made on a quantum system, the state of the system is altered.

Let’s consider the following example. Suppose we want to make a measurement regarding a physical quantity A on a system which is currently in state X. We then propose that A = μ where μ is the result of the measurement. In classical physics, this proposition is true for the system in the state X. In quantum mechanics however, the proposition is true for the system in state Y, the state of the system after the measurement took place. Unlike classical logic, it is not necessary for a proposition to have any truth value in a quantum system in its current state. It may be the case that the proposition acquires its value in another state, for example in a state after some measurement. We therefore see that it is not possible to reason about quantum mechanics with classical logic but instead we have to use the logic that takes quantum ideas into consideration.

Five significant differences between classical and quantum logic are described below:

Quantum disjunction

The classical disjunction (OR) of two propositions A and B states that either A is true, or B is true, or both are true. On the contrary, in quantum logic it is possible for the disjunction A V B to be satisfied in some state while neither A or B hold by themselves. This is particularly interesting when A and B describe different values for the same physical quantity.

Suppose a particle’s spins is measured in two directions, x-axis (up or down ) y-axis(left or right). By the principle of uncertainty the value of y-axis spin will be indeterminate from the state after verifying the spin in x-axis. In quantum logic if a proposition is indeterminate then its truth value is false.

So if we have the following propositions:

A: spin y is left
B:spin y is right
After verifying spin x, both propositions A and B are indeterminate
A V B = False V False
However we know that the proposition A V B (either spin y is left or right) must be true
A V B = True

Failure of distributive law

The distributive law in algebra states that:

a x (b + c) = a x b + a x c

The law is valid also in classical propositional logic:

A V (B Ʌ C) = (A V B) Ʌ (A V C)
A Ʌ (B V C) = (A Ʌ B) V (A Ʌ C)

However this is not the case in quantum logic. This is because a quantum disjunction can be true even if both of the elements of the disjunction are not true. These correspond to the common case in quantum mechanics, where a pair of properties is semantically defined but each of its members is necessarily undefined.

For example, suppose we have a particle that moves in a certain direction across an axis (e.g. x -axis). By the principle of uncertainty the value of position will be indeterminate from the state after verifying the direction of the move, so the following propositions P Ʌ Q and P Ʌ R are not necessarily true.

P: The direction of move of the particle is positive
Q: The position of the particle is < k
R: The position of the particle is >=k
P Ʌ (Q V R) = P Ʌ True = P
(P Ʌ Q) Ʌ (P Ʌ R) = False V False = False

The two expressions above are not equal.
Hence the distributivity law fails

Failure of commutativity

In logic, as in mathematics, a binary operation is commutative if and only if the result does not depend on the order of the operands. This property holds when applied over conjunctions or disjunctions in classical logic. For example, if we change the order of A and B in (A Ʌ B) or in (A V B), the result of the function does not change.

However, in quantum logic, the commutative law does not stand. This is simple to explain. Let's take into consideration the basic rules of quantum computing that have been established since now. In quantum mechanics a system has a state at a specific time. Any measurement one makes to the system will change its state. A further measurement will be done to the altered state, making the result of this measurement invalid for the initial system. To understand the implications of this principle, consider two physical properties X and Y (e.g. position and energy of a particle) and two quantum operators Xop and Yop that are applied on these properties respectively (e.g. is position within an area, energy is over certain value). Then try to assign them on a function F (could be the logical and/or) first in order F Xop Yop and then F Yop Xop. In the first case first we verify Xop, and this changes the systems state to a new state (S1), then when verifying Yop we are verifying it in state S1 and not the initial state and some result, in the second case we verify Yop and this changes the system into a new state (S2), then when verifying the Xop we are verifying it in state S2 and this may give a different result.

1st case: F Xop Yop: Xop verified when system in initial state, Yop verified when system in S1 state
2nd case: F Yop Xop: Yop verified when system in initial state, Xop verified when system in S2 state

Failure of excluded middle (proof by contradiction)

Propositions in classical logic can be true or false. These are the only possibilities, i.e. there is no middle value. For example consider the following proposition:

Socrates is mortal.

By the law of excluded middle we can say that either this proposition is true or its negation is true.

i.e. the disjunction

Socrates is mortal or Socrates is not mortal

is always true. The middle position, that Socrates is neither mortal nor immortal, is excluded by logic. As a result, if one wants to prove the truthfulness of a proposition, all he has to do is to prove the falseness of its exact opposite proposition. This was first proposed by Aristotle.

Since quantum logic is not two-valued logic, the law of excluded middle does not hold. For example, consider the unit of quantum information, the qubit. Unlike the normal bit which is Boolean (value 1 or 0 only) the qubit can be in states |0>, |1> or in a superposition of those two. Since a qubit can take many values, this means that if you prove that a qubit does not take the value 1 you cannot know for sure that it takes the value 0, as it can take any other superposition of 1 and 0.

Non-monotonicity

Classical logic is monotonic. This means that in the case where we have more information (true statements or assumptions), we can still make the deductions that could be made with less information. For example, suppose we know that propositions A and A -> B (A implies B) are both true in a situation. From these we can deduce that B is also true. Now if we know that C is true in addition to A and A -> B in that same situation, we can still deduce that B is true as well.

Quantum logic on the other hand is non-monotonic because it is affected by Heisenberg’s Uncertainty Principle. Let’s again consider an example. Suppose we measure the energy (E) of an electron in an atom and we find that it is equal to μ. This ‘proves’ the proposition E = μ. We now measure the position (P) of the electron and find out that it is equal to k. This in turn proves the proposition P = k. In classical logic we could now say that we know both E and P, and that any subsequent measure of E will still result in μ. In quantum logic however this is not the case. A further measurement on E will return a value different from μ. This is due to the fact that by measuring P we got more information (regarding the electron’s position) but we have invalidated the original result about energy. This conclusion is based on the fact that there exists an uncertainty relation between energy and position, which suggests that the more we know about one of the two quantities, the less we know for sure about the second. This, as far as logic is concerned, is non-monotonicity. The extra information about position changed the validity of what we knew about energy.

Direct arguments
This is a proof of ((p → q) → p) → p (known as 'Peirce’s law').

Take an arbitrary situation. If p is true in this situation, then ((p → q) → p) → p is true, since any formula A → B is true when B is true. We are done. If not, then p must be false in this situation. So p → q is true, because A → B is true when A is false. So (p → q) → p is false, because the LHS is true and the RHS false: A → B is false when A is true and B false. So ((p → q) → p) → p is true, because A → B is true when A is false. We are done again, and finished.

This was an argument by cases: p true, or p false. They are exhaustive: this is known as the ‘law of excluded middle’. However as we know this law doesn’t hold in quantum logic, law of included middle holds instead. So this proof isn't exhaustive for quantum logic. The case of p = any superposition between 1 and 0 (True and False) should be checked.

Use of Equivalences
In classical propositional logic we can proof that (p → r) ∧ (q → r) is equivalent to p ∨ q → r:
1. (p → r) ∧ (q → r) is equivalent to:
2. (¬p ∨ r) ∧ (¬q ∨ r) (using p → q equivalent to ¬p ∨ q),
3. (¬p ∧ ¬q) ∨ r (using distributivity of ∧ over ∨),
4. ¬(p ∨ q) ∨ r (using De Morgan law),
5. p ∨ q → r (using p → q equivalent to ¬p ∨ q again).

In quantum logic step 3 is not valid since it uses distributivity that it does not hold.

Now consider the following example:

If we see wet grass the most likely explanation is that it rained. However, this explanation has to be retracted when learning that the real cause of the grass being wet was a sprinkler. Since the old explanation (it rained) is retracted because of the addition of a piece of knowledge (a sprinkler was active), any logic that models explanations is non-monotonic.

W: Grass is wet
R: It rained
S: sprinkler was active

1. W     given
2. W → R     given
3. R     →E(1,2)

However if we add another given into our system

1. W     given
2. W → R     given
3. S     given
4. R     →E(1,2)

In classical monotonic logic we can still deduce that it rained. In quantum logic, a non-monotonic logic, this is not possible. The extra information that we have about the sprinkler invalidates our previous inference.
